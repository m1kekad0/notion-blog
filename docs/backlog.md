# バックログ / スコープ外機能

`requirements_definition.md` に記載されているが、現時点では実装していない機能の一覧。
「廃止」ではなく「先送り」として管理する。

---

## スコープ外（未実装）

### キーワード検索

- **要件**: 記事タイトル・本文でのキーワード検索機能
- **現状**: 未実装。`Post` 型にも検索関連フィールドなし
- **実装候補**: Notion Search API はクエリ文字列をサポートしており、
  新インフラなしで基本的な検索が実現可能
- **優先度**: 低（個人ブログのスケールでは記事一覧 + タグで代替可能）

---

### カテゴリ別フィルタ

- **要件**: カテゴリ単位での記事一覧表示
- **現状**: 未実装。`Post` 型に `category` フィールドなし。
  Notion データベーススキーマにも `category` プロパティなし
- **備考**: タグ（`tags`）による分類は実装済み。
  カテゴリが必要な場合は Notion に `category` プロパティを追加し、
  `Post` 型と `getPosts()` のマッピングを更新する
- **優先度**: 低

---

### OGP 画像（og:image）

- **要件**: 記事カードおよび SNS シェア時のアイキャッチ画像表示
- **現状**: `metadata` に `og:image` が設定されていない。
  記事一覧カードにもカバー画像表示なし
- **実装候補**:
  - Notion ページのカバー画像を `og:image` に使用する
  - `@vercel/og` や Cloudflare Workers での動的 OGP 画像生成
- **優先度**: 中（SNS シェア時の見栄えに影響）

---

### Mermaid 図のレンダリング

- **要件**: Mermaid 記法によるダイアグラムの表示
- **現状**: 未実装。`package.json` に Mermaid 関連パッケージなし。
  Mermaid フェンスコードブロックはプレーンテキストとして表示される
- **実装候補**: `react-mermaid2` などのパッケージ追加と
  `ReactMarkdown` の `code` コンポーネントカスタマイズ
- **優先度**: 低

---

### 無限スクロール

- **要件**: ページネーションまたは無限スクロールによる記事一覧
- **現状**: ページネーション（`?page=N`、1ページ10件）で実装済み
- **備考**: 要件の「または」のうちページネーションを選択済みのため、
  無限スクロールは先送り
- **優先度**: 低

---

### Notion 画像の Cloudflare Worker プロキシ（S3 署名付き URL 失効の抜本解決）

- **背景**: Notion がホストする S3 署名付き画像 URL は約 1 時間で失効する。
  現状は `revalidate=300` + `Cache-Control: s-maxage=300, stale-while-revalidate=0` で
  キャッシュ期間を抑制しているが、構造的な解決ではない
- **実装候補**: Cloudflare Worker を画像プロキシとして実装し、
  Notion S3 URL を Worker 経由でフェッチ・キャッシュすることで URL 失効問題を根本解決する。
  または Cloudflare Images サービスを利用して画像を転写・永続化する
- **優先度**: 中（現状の暫定対策で個人ブログスケールは許容範囲内）

---

### Search API 全件取得のスケーラビリティ対応

- **背景**: `getPosts()` は `notion.search` で全件取得しクライアントサイドでフィルタリングしている。
  記事数が 100 件を超えるとページネーションループが増加し、ISR revalidate 毎の
  Notion API コール数が増大する。また、ワークスペースにブログ以外のページが蓄積すると
  フェッチ量がさらに増える
- **実装候補**: `@notionhq/client` の `databases.query` が将来復旧した際に移行する。
  またはワークスペースを分離しブログ専用にする
- **優先度**: 低（記事数が 100 件を超えた段階で対応を検討する）
